# Проект «Витрина данных для маркетплейса “ВсёТут”»

## \Цель
Подготовить аналитическую витрину клиентских данных для запуска ML-моделей (сегментация, прогноз оттока, персонализация) и продуктовой аналитики на основе заказов пользователей маркетплейса.

Проект выполнен в рамках курса «Специалист по Data Science» на Яндекс Практикуме (январь 2026).

---

##  Что сделано

### 1. Создана витрина `product_user_features`
- Отфильтрованы заказы со статусами **«Доставлено»** и **«Отменено»**.
- Выделены **3 региона с наибольшей активностью**: Москва, Санкт-Петербург, Новосибирск.
- Для каждой пары «пользователь-регион» рассчитаны ключевые метрики:
  - жизненный цикл (lifetime), количество и стоимость заказов;
  - доля отмен, использование промокодов и рассрочки;
  - бинарные признаки: использовал ли денежный перевод, отменял ли заказ.

### 2. Решены 4 ad hoc-задачи
1. **Сегментация пользователей** по количеству заказов, выявлен доминирующий сегмент «1 заказ».
2. **Ранжирование** по среднему чеку среди активных пользователей (≥3 заказа).
3. **Сравнение регионов** по поведению: Москва лидирует по числу клиентов, но не по среднему чеку.
4. **Анализ новичков 2023 года** по месяцу первого заказ, пик регистрации — ноябрь–декабрь.

---

##  Ключевые инсайты

- **78% пользователей совершают только 1 заказ**, высокий риск оттока.
- Чем выше лояльность (число заказов), тем **ниже средний чек**, возможно, это «экономные» постоянные покупатели.
- **Москва** крупнейший регион по числу клиентов, но **Санкт-Петербург** показывает **самый высокий средний чек** (3620 руб.).
- Использование **денежных переводов (~20%)**, **рассрочки (~50%)** и **промокодов (~4%)** стабильно по регионам.
- Новые пользователи в конце 2023 года демонстрируют **более высокую активность** (до 12 дней lifetime).

##  Рекомендации
- Запустить **кампанию удержания** для сегмента «1 заказ» (например, welcome-бонус при втором заказе).
- Предложить **персонализированные предложения** пользователям из Санкт-Петербурга с высоким чеком.
- Использовать **стабильность метрик по регионам** как основу для единых ML-моделей без сильной региональной сегментации.
